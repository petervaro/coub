Questions and Answers

Why is it called 'Coublet'?

The name Coublet has two meanings. It comes from the mix of the two words 'coub' and 'goblet' — as a symbolic vessel for all the fantastic coubs out there. It is also a mixed expression of the words 'coub' and 'applet' — as it is a desktop client for a web-service.

How does it work?

Coublet right now supports 4 streams: 'featured', 'newest', 'random' and 'hot'. You can click on the bottom menu, to choose any of them. If auto-update is supported by the given stream, Coublet will check if new coubs are available to download and will pull them.


If you scroll down to the bottom, and then scroll again a bit more, it will load more coubs. If you scroll up to the top and then scroll again a bit more, it will synchronise the coubs from the server and will keep your stream up to date.


You can left-click on a post to play or pause it. By right clicking on a coub you can mute or unmute it. Double clicking with the left mouse button, it will open the coub in your default browser. Double clicking with the right mouse button it will reset the coub. If a post is not visible — maybe you switched to another stream or scrolled away from the post — the coub will be stopped.

Why can't I log in?

At the moment coub.com doesn't have an official public API — which means, the authentication is not publicly available for 3rd party app developers. (As far as I know, they are working on one right now. Once it will be released Coublet will support personal streams, likes and recoubs as well.)

What platforms Coublet is supporting?

Coublet does not using any platform specific language or technology. It is implemented in Python and using Qt and both are supporting the vast majority of the operating systems. Therefore Coublet supports Windows, Mac and Linux too.

Will it ever be free?

Yes it will, personally I believe in the free software movement. However, I will add a donation service soon (Flattr maybe?) and any support will be appreciated and more than welcome.

How can I install it?

Coublet is under heavy development, which means there are no bundles (packed executable binaries) available at the moment. However, you can build the project on your own if you want to test it. To do that, you will need the followings:


Python  ->  3.4.1+
PyQt5   ->  3.5.1+


You may also want to disable the developer mode (as cutils is not included in the repository). All you have to do, is open main.py and set the DEV flag to False.


Note: Although there is a setup.py in the project folder, it is not working yet. In the near future, Coublet will use cx_Freeze to build the standalone applications on all platforms.

If I want to test it, where can I download it from?

You can download it from the the GitHub repository of the project.
